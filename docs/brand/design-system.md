# Design System - AuditData AI

## Principio visual

AuditData AI debe transmitir innovacion, orden, claridad, tecnologia, confianza y escalabilidad.

## Paleta

- Azul electrico `#0066FF`: accion principal.
- Azul oscuro `#0052CC`: hover y jerarquia secundaria.
- Cian `#00D4FF`: acentos analiticos, tags y datos destacados.
- Negro `#0A0A0F`: fondo principal.
- Gris oscuro `#12121A`: tarjetas y superficies.
- Blanco suave `#F0F0F5`: texto principal.
- Gris metalico `#9090A0`: texto secundario.

## Componentes base

- Hero: presenta objetivo de la herramienta.
- Status Card: informa estado del sistema.
- Step Card: muestra avance del proceso.
- Panel: agrupa contenido funcional.
- Dropzone: entrada principal de datasets.
- Button: acciones primarias, secundarias y fantasma.
- Metric Card: indicadores numericos.
- Table: hallazgos estructurados.

## Criterio de interaccion

Las microinteracciones deben ser sobrias:

- Hover con desplazamiento minimo.
- Estados de carga claros.
- Feedback textual en el estado del sistema.
- Sin animaciones decorativas excesivas.
